自动化工具Grunt和Gulp

2014-09-02 develop 浏览 留言互动

伴随着随着nodejs的出现，越来越多的前端自动化工具出现在了人们的视野里，比如：grunt、gulp、Yeoman、FIS、bower等。自动化工具的确大大解放了前端工程师们的双手，比如可以自动检测文件改动自动编译、压缩、刷新浏览器等，让前端工程师们可以更专心的coding，而不是传统的不断的重复着F5。

2014年上半年一不小心接触了Grunt，然后也花了几天去小看了一下相关的资料。然后最关键的其实就是那两个配置文件了，即：Gruntfile.js和package.json。

再接着没多久，就又接触了gulp。grunt是同步的，而gulp是异步的。相对而言，gulp也显得更简洁单一和高效吧，是一种stream工作流模式。在实际工作中，grunt使用了一段时间，然后也许是受潮流的推动和PC机上（公司的台式机）运行一段时间后的反应略慢的原因，最后又走向了gulp。

gulp和grunt可以说是大同小异，本质上都是一种效率工具。不过其实现的原理和运行的效率还是有点差别的。其实在目前而言，两种工具都是不错和足以应付我们的常规工作的。

这两种自动化工具都依赖于两个文件，即一个Gruntfile.js(grunt)/Gulpfile.js(gulp)和package.json。网络上的教程其实也挺多的，具体的可以直接百度或者谷歌吧。那么下面就自己的情况，做一个备份吧。

这两种自动化工具使用起来都很简单，在安装好nodejs、ruby、grunt/gulp后，进入有grunt/gulp配置文件的文件夹，然后在CMD/终端里运行自定义的命令就ok。

一些说明：

  • nodejs是基础环境，ruby是因为使用sass而需要的环境。

  • windows上，可以进入你想要使用命令窗口的文件夹里，然后Shift+鼠标右键选择在此处打开命令窗口（W）。Mac上就简单多了，直接cd path。

  • 对于安装而言，本质上都是npm的使用。根据官方文档提供的模块名称安装即可，比如全局安装grunt是（全局安装，在Mac下需要sudo）：

npm install -g grunt-cli // install后面的命令顺序随意，-g是-global的简写，下同

对应的gulp也一样：npm install -g gulp

对于插件的安装本质上和安装grunt和gulp自身其实都差不多，都是安装npm模块。

npm install --save-dev gulp // 安装的信息会写入到package.json中的devDependencies里，方面下次一键安装。不带--save*这个的则只安装不写入配置文件里

–save和–save-dev感觉在实际使用中，似乎没什么差别。–save则是写入到package.json中的dependencies里。

  • 对于配置文件而言，也是需要根据官方提供的格式，添加自己需要的模块插件和定义自己需要的任务。最后进入项目文件里，使用命令npm install一键安装或者自定义安装新的模块即可。

###关于gulp的补充：

几个主要部分简单的说一下：

  • src表示的是文件来源，以及可重置文件夹起始位置。
  • pipe表示要引入的任务内容，流水式作业。
  • gulp.dest：表示输出的目录。

或者这样来说：

  • gulp.task('taskname', [ taskDep1, taskDep2 ],taskContentFunc)定义一个任务,声明它的名称, 任务依赖, 和任务内容.
  • gulp.src( file[s] )读取文件,参数为文件路径字符串或数组, 支持通配符.
  • gulp.dest( destPath )写入文件, 作为pipe的一个流程.文件夹不存在时会被自动创建.
  • gulp.watch(files, [taskDep, taskDep2])监控文件,执行任务.
